# divkix.me Aesthetic Redesign Plan

> **Direction:** "Engineering Workbench" — brutalist-precise, raw materials, perfect execution.
> **Reference:** Stripe engineering blog meets well-configured Neovim meets NYT editorial layout.

---

## Current State Diagnosis

The site looks like it was generated by an AI in 2024. Every design decision screams "default":

- **Fonts:** Inter + Space Grotesk — THE most overused AI-generated font combo in existence
- **Colors:** Purple primary + pink accent gradient (oklch 264.5/310 hue) — same gradient on 10,000 other portfolios
- **Layout:** Center-aligned everything, uniform `py-20`, symmetric grids — zero visual tension
- **Animations:** `fade-in-up` on everything, aurora gradient blobs that add nothing
- **Cards:** Standard shadcn bordered rectangles with hover elevation — cookie-cutter
- **Hero:** Giant centered name + gradient text + single button — seen 10,000 times

**The ONE element with personality:** Skills terminal section — CRT scanlines, phosphor-green text, typing cursor. That's the DNA to amplify.

---

## 1. Typography

**Kill:** Inter (body) + Space Grotesk (display)

**Replace with:**

| Role | Font | Package | Why |
|------|------|---------|-----|
| **Display/Headings** | Instrument Serif | `@fontsource/instrument-serif` | Editorial serif on a dev portfolio is a power move. Maximum tension against terminal elements — warmth vs rawness. Unexpected, memorable. |
| **Body** | Geist Sans | `@fontsource/geist-sans` | Vercel's own font. Clean, technical, modern. Purpose-built for interfaces. |
| **Monospace** | Geist Mono | `@fontsource/geist-mono` | Used everywhere: terminal, section labels, metadata, code. Replaces runtime Google Fonts JetBrains Mono import. Same design family as body font. |

### CSS Config

```css
@theme inline {
  --font-sans: 'Geist Sans', ui-sans-serif, system-ui, sans-serif;
  --font-display: 'Instrument Serif', Georgia, serif;
  --font-mono: 'Geist Mono', ui-monospace, monospace;
}
```

### Typography Principles

- Serif headings + monospace labels = distinctive tension
- Section labels in monospace above every heading: `// 01 — projects`
- JetBrains Mono Google Fonts runtime import in SkillsTerminal replaced with bundled Geist Mono
- `text-wrap: balance` on headings, `text-wrap: pretty` on paragraphs

## 4. Layout Philosophy

### Kill the Symmetry. Build Rhythm.

- **Asymmetric hero** — left-aligned, not centered
- **12-column editorial grid** for select sections
- **Monospace section labels** above every heading: `// 01 — projects`
- **Variable section spacing** (rhythm, not metronome):

| Section | Padding | Rationale |
|---------|---------|-----------|
| Hero | 100vh | Full viewport presence |

## 5. Section-by-Section Redesign

### 5.1 Navbar: "Command Bar"

**Current:** Generic sticky nav, glass-surface backdrop blur, image logo, plain text links with tiny underline indicator.

**Redesign:**

- Monospace nav labels in Geist Mono
- Text logo `divkix_` in monospace instead of image swap
- Active indicator: amber underline that **slides** between items (layout animation or CSS translate), not fade-in
- On scroll: thin gradient `border-bottom` (amber to teal, 1px) instead of shadow/blur
- Mobile: full-screen overlay with large typography and staggered entrance (each link slides in from right with 50ms delay), replace stock Sheet sidebar

**Techniques:**

- `border-image: linear-gradient(to right, var(--primary), var(--accent)) 1` for bottom border
- CSS `view-transition-name` per nav item for smooth indicator sliding
- Container queries for responsive compression without media queries

### 5.2 Hero: "Editorial Terminal Hybrid"

**Current:** Centered text stack, animated gradient on last name, two faint blurred circles, single "View Projects" button, bouncing chevron.

**Redesign:**

- **Left-aligned** (not centered), asymmetric layout
- Monospace label `@divkix` with typewriter cursor positioned above the name
- Name in massive Instrument Serif (8xl+), stacked vertically across 2-3 lines
- Tagline offset right, creating visual tension
- Two CTAs: "View Projects" (primary, hard shadow) + "Read Blog" (outline with arrow)
- Social icons: vertical stack on left edge or bottom-left
- Background: dot grid only, no gradient blobs
- Optional right panel (40%): interactive terminal snippet showing `whoami` output

**Techniques:**

- CSS Grid `grid-template-columns: 3fr 2fr` for asymmetric split (if using terminal panel)
- Typewriter: CSS `steps()` animation + `ch` units for width
- `@starting-style` for terminal panel entrance
- Staggered reveal with **varied easing** per element (not all ease-out)

### 5.3 Highlights: "Raw Typography Stats"

**Current:** Three identical stat cards, symmetric grid, glass-surface, rotating border on hover.

**Redesign:**

- **No cards.** Just raw, massive typography directly on the page.
- Asymmetric layout: main stat (250K+ users) gets huge full-width treatment (massive number, small label). Other two stats stacked smaller to the right.
- Numbers in 7xl+ Geist Mono, labels in small Geist Sans below
- Count-up animation stays (it's good)
- `font-variant-numeric: tabular-nums` for clean alignment during count-up

**Techniques:**

- CSS Grid `grid-template-columns: 2fr 1fr` with `grid-template-rows: 1fr 1fr` for stacked pair
- Scroll-driven reveal animation
- No borders, no cards, no glass — just typography

### 5.4 Projects: "Editorial Showcase"

**Current:** Bento grid, TiltCard 3D effect, filter chips centered, stock badges/buttons, no imagery.

**Redesign:**

- **Featured projects** (with "Live" links): full-width hero card with split layout — info left, browser mockup right (styled div with rounded corners, fake address bar, gradient fill using project's accent color)
- **Regular projects**: tighter 2-column grid, horizontal card layout (image left, text right)
- **Kill the 3D tilt** — gimmicky
- Hard shadow cards, thin border, 4px colored accent bar at top of each card
- Filter: left-aligned segmented control (sliding indicator) instead of centered chips
- Tags: monospace Geist Mono, smaller, color-coded per language
- Tech stack icons as grayscale that colorize on hover

**Techniques:**

- Featured card: `grid-template-columns: 1fr 1fr`
- Browser mockup: styled div with rounded-t corners, fake address bar div, gradient fill
- Accent bar: `border-top: 4px solid var(--card-accent)`
- Segmented control: `position: relative` container with sliding `::before` via `translateX`
- Card entrance: stagger with IntersectionObserver + CSS `@starting-style`

### 5.5 Experience: "Git-Log Timeline"

**Current:** Bento grid, hover-to-reveal highlights (mobile users never see them), barely perceptible scale on hover.

**Redesign:**

- **Vertical timeline** with amber-colored line on the left
- Nodes at each company, cards extending right
- Current role: pulsing green node (keep the ping animation)
- Past roles: static muted nodes
- Inside each card: company name bold, with mini nested timeline showing position progression
- **All highlights always visible** — no more hover-to-reveal
- Education: separate horizontal banner at top with distinct treatment
- Desktop: alternating sides (odd right, even left) using `:nth-child(even)` with `flex-direction: row-reverse`

**Techniques:**

- Timeline: `border-left: 3px solid var(--primary)` with `::before` pseudo-element circles
- Cards: `margin-left: 2rem` with `::before` horizontal connector line
- Scroll-triggered: IntersectionObserver, cards slide in from alternating sides
- Current node: `box-shadow: 0 0 0 4px oklch(var(--primary) / 0.2)` with pulse animation

### 5.6 Skills: "Full Hacker Terminal" (KEEP + AMPLIFY)

**Current:** Best section on the site. 4 terminal windows in a 2x2 grid, CRT scanlines, blinking cursor, block-character skill bars.

**Redesign:**

- **Full-bleed, full-width** — breaks out of container
- True black background `oklch(0.08 0 0)`
- **Single terminal window** with tabs per category (Languages | Frameworks | Databases | Tools) instead of 4 separate windows
- Typing animation: `$ divkix --list-skills --category=languages` then output appears line by line
- Stronger CRT effects: scanline opacity 0.03 to 0.06-0.08, screen vignette (`box-shadow: inset 0 0 60px rgba(0,0,0,0.3)`), subtle screen flicker on initial load (200ms)
- `text-shadow: 0 0 2px currentColor` on all terminal text for CRT phosphor glow
- Replace Google Fonts JetBrains Mono with bundled Geist Mono
- Terminal chrome accent color shifts to amber (title bar area)
- Section transition: gradient divider that fades from page bg to terminal black

**Techniques:**

- Tabs: `<button>` with `aria-selected`, `role="tab"` for a11y
- Typing: `@keyframes typing` with `width: 0 to 100%` + `steps()` + `overflow: hidden; white-space: nowrap`
- CRT curvature: `border-radius: 8px` + inset box-shadow for vignette
- Flicker: `@keyframes flicker { 0% { opacity: 0.8 } 5% { opacity: 1 } 10% { opacity: 0.85 } 15% { opacity: 1 } }`

### 5.7 Contact: "Terminal Mail Client"

**Current:** Standard centered form in glass card. Stock shadcn form elements.

**Redesign:**

- **Terminal aesthetic** extending from Skills — dark background, monospace everything
- Card becomes a terminal window with macOS-style title bar dots (reuse terminal component)
- Input fields: no borders, just underlines, Geist Mono font, `>` prompt character before each field
- Submit button: `$ send --message` styled as terminal command (monospace, outline, amber/green)
- Below form: "or reach me at" section with social links as terminal output: `-> github.com/divkix`
- Success state: `[OK] Message sent. Expect a response within 24h.` in green terminal text (typewriter effect)

**Techniques:**

- Extract shared terminal window component from SkillsTerminal
- Input: `background: transparent; border: none; border-bottom: 1px solid oklch(1 0 0 / 20%); font-family: 'Geist Mono'`
- Prompt: `::before { content: '> '; color: oklch(0.7 0.2 140) }` on each field wrapper
- Button: `font-family: 'Geist Mono'; border: 1px solid oklch(0.7 0.2 140); color: oklch(0.7 0.2 140)`

### 5.8 Footer: "Terminal Status Bar"

**Current:** Single-line footer, copyright + links + social icons. Forgettable.

**Redesign:**

- Thin dark strip matching terminal sections (`background: #1a1a2e`)
- Left: `divkix@portfolio:~$` prompt with subtle cursor
- Center: navigation links styled as terminal commands
- Right: `status: online` with green dot + year
- "Open Source" link becomes clickable `[src]` badge
- Small text (12-13px), Geist Mono, high information density

**Techniques:**

- `font-family: 'Geist Mono'`, `font-size: 0.8rem`

## 6. Animation Strategy

### Kill

- `fade-in-up` on everything (lazy, uniform)
- Aurora gradient drift
- Gradient text shimmer (`gradient-text-animated`)
- 3D tilt on project cards

### Replace With

| Element Type | Animation | Technique |
|-------------|-----------|-----------|
| Section reveals | Scroll-driven upward translate | CSS `animation-timeline: view()` with IntersectionObserver JS fallback |
| Text blocks | Clip-path reveal from left | `clip-path: inset(0 100% 0 0)` to `inset(0)` |
| Cards | Scale 0.95 + opacity fade | CSS `@starting-style` for enter animations |
| Numbers/stats | Count-up (keep existing) | IntersectionObserver + requestAnimationFrame |
| Terminal content | Typing + line-by-line reveal | CSS `steps()` + staggered delays |
| Buttons hover | Snap transform (not float) | `translateY(-1px)` + hard shadow appearing |
| Link hover | Underline grows from left | `scaleX(0)` to `scaleX(1)` on `::after` |
| Section dividers | Thin gradient line or ASCII art | `// ----------- //` monospace separators |
| Nav indicator | Sliding pill | Layout animation or CSS translate |

### Libraries

- **Motion** (Framer Motion successor) for React island component animations
- **CSS scroll-driven animations** for lightweight, non-React Astro elements
- Progressive enhancement: CSS scroll-driven with JS IntersectionObserver fallback

### Scroll-Driven Animation Base

```css
.reveal-on-scroll {
  animation: reveal-up linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 30%;
}

@keyframes reveal-up {
  from {
    opacity: 0;
    transform: translateY(2rem);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@supports not (animation-timeline: view()) {
  .reveal-on-scroll {
    animation: none;
    opacity: 1;
  }
}
```

## 7. Two-Act Narrative

The page tells a story through its aesthetic:

### Act 1 — Professional Polish (Hero -> Highlights -> Projects -> Experience)

- Instrument Serif headings, clean Geist Sans body
- Warm cream / warm dark backgrounds, amber accents
- Editorial layouts, hard shadows, dot grid texture
- Message: "I'm polished, I have taste, I ship real products"

### Act 2 — Under the Hood (Skills -> Contact -> Footer)

- Full terminal aesthetic: dark background, Geist Mono everywhere
- CRT effects, typing animations, phosphor green text
- Terminal-styled forms, status bar footer
- Message: "Under the hood, I'm deeply technical"

### The Transition

At the Skills section boundary, the background snaps from page color to true black `oklch(0.08 0 0)`. Font shifts to monospace. CRT effects activate. The user "enters the terminal."

This deliberate contrast says: *"I present professionally, but under the hood, I'm deeply technical."*

## 9. Technical Notes

### Dependencies to Add

```bash
bun add @fontsource/instrument-serif @fontsource/geist-sans @fontsource/geist-mono
```

### Dependencies to Remove

```bash
bun remove @fontsource-variable/inter @fontsource-variable/space-grotesk
```

### Files Requiring Changes

**Global/Config:**
- `src/styles/globals.css` — color variables, font config, background, shadows, border-radius
- `src/styles/animations.css` — new keyframes, remove old ones
- `src/layouts/BaseLayout.astro` — font imports, replace aurora divs with dot grid

**Components:**
- `src/components/sections/Hero.astro` — full layout restructure
- `src/components/sections/Highlights.tsx` — remove cards, raw typography
- `src/components/sections/Projects.tsx` — editorial layout, kill tilt, accent bars
- `src/components/sections/experience/ExperienceBento.tsx` — timeline layout
- `src/components/sections/skills/SkillsTerminal.tsx` — single terminal, tabs, enhanced CRT
- `src/components/sections/Contact.tsx` — terminal form
- `src/components/shared/Navbar.tsx` — command bar aesthetic
- `src/components/shared/Footer.astro` — terminal status bar

### Shared Components to Extract

- `TerminalWindow` component (shared between Skills, Contact, Footer)
- `SectionLabel` component for monospace `// 01 — section` headers
- `useScrollReveal` hook for consistent IntersectionObserver reveals

### Accessibility Considerations

- CSS animations and transitions respect `prefers-reduced-motion: reduce`; JS-driven effects (terminal typing, typewriter success) detect the preference via `matchMedia` and skip animation when enabled
- Terminal sections maintain sufficient color contrast (WCAG AA minimum)
- Tab-based Skills terminal uses proper `role="tablist"`, `role="tab"`, `aria-selected`
- Form inputs in terminal style maintain proper `<label>` associations
- Scroll-driven animations degrade gracefully (content visible without animation)
